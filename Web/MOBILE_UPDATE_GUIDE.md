# 📱 手机更新比赛数据指南

## 🎯 使用 GitHub Mobile App 更新比分

### 步骤

1. **下载 GitHub Mobile App**
   - iOS: [App Store](https://apps.apple.com/app/github/id1477376905)
   - Android: [Google Play](https://play.google.com/store/apps/details?id=com.github.android)

2. **登录并打开项目仓库**

3. **编辑比分文件**
   - 打开 `Web/public/data/scores.json`
   - 点击右上角编辑按钮
   - **只需要修改数字！**
   - 点击 "Commit changes"

4. **等待自动部署**
   - GitHub Pages 会自动触发构建
   - 约 1-2 分钟后网站更新
   - 积分榜会自动刷新（每30秒）

## 📝 JSON 格式说明

### scores.json 格式（超简单！）

```json
{
  "matches": [
    {
      "id": "R1-1",
      "info": "第1轮 | 女双(WD) | A队(苏苏🌟/舒婵) vs B队(夏天吃黄瓜/🍊姐)",
      "games": [
        [21, 19],
        [18, 21],
        [21, 15]
      ]
    }
  ]
}
```

### 字段说明

- **id**: 比赛ID（**不要修改**，保持原样）
- **info**: 比赛信息（**不要修改**，仅作参考）
  - 包含：轮次、项目类型、参赛选手
  - 编辑时可以参考，避免搞混比赛
- **games**: 比分数组，**只需要改数字**
  - `[]` = 比赛未开始
  - `[[21, 19]]` = 一局，A队21分，B队19分
  - `[[21, 19], [18, 21], [21, 15]]` = 三局

### 比分格式

每局格式：`[A队得分, B队得分]`

**示例：**
- `[21, 19]` → A队21分，B队19分，A队胜
- `[18, 21]` → A队18分，B队21分，B队胜
- `[21, 15]` → A队21分，B队15分，A队胜

## ✏️ 更新示例

### 示例1：更新第一场比赛（三局两胜）

**修改前：**
```json
{
  "id": "R1-1",
  "info": "第1轮 | 女双(WD) | A队(苏苏🌟/舒婵) vs B队(夏天吃黄瓜/🍊姐)",
  "games": []
}
```

**修改后：**
```json
{
  "id": "R1-1",
  "info": "第1轮 | 女双(WD) | A队(苏苏🌟/舒婵) vs B队(夏天吃黄瓜/🍊姐)",
  "games": [
    [21, 19],
    [18, 21],
    [21, 15]
  ]
}
```

**说明：**
- `info` 字段显示这是第1轮的女双比赛
- A队选手：苏苏🌟、舒婵
- B队选手：夏天吃黄瓜、🍊姐
- 比分：A队2:1获胜（第一局21:19，第二局18:21，第三局21:15）

### 示例2：两局结束

```json
{
  "id": "R1-1",
  "info": "第1轮 | 女双(WD) | A队(苏苏🌟/舒婵) vs B队(夏天吃黄瓜/🍊姐)",
  "games": [
    [21, 15],
    [21, 18]
  ]
}
```

**结果：** A队2:0获胜（两局直接结束，无需第三局）

## 💡 重要提示

✅ **只需要修改数字**，其他格式保持不变  
✅ `info` 字段是注释信息，**不要修改**，仅作参考  
✅ 每局比分格式：`[A队得分, B队得分]`  
✅ 空数组 `[]` 表示比赛未开始  
✅ 系统会自动计算胜负、积分和排名  
✅ 积分榜会自动刷新（每30秒）  
✅ 比赛进行中时不显示排名特效（无冠亚季军标签）  
✅ 编辑时可以参考 `info` 字段确认比赛信息，避免搞混

## ⚠️ 注意事项

1. **JSON 格式**：确保格式正确，否则网站可能无法加载
2. **数字范围**：比分通常在 0-30 之间
3. **提交频率**：建议每场比赛结束后统一更新
4. **数据备份**：GitHub 会自动保存历史版本，可以随时回退
5. **比赛状态**：在 `standings.ts` 中设置 `tournamentStatus: 'ongoing'` 启用实时积分榜

---

## 🔧 比赛状态设置

编辑 `Web/src/data/standings.ts`：

```typescript
// 比赛状态：'upcoming' | 'ongoing' | 'finished'
export const tournamentStatus: 'upcoming' | 'ongoing' | 'finished' = 'ongoing';
```

- `'upcoming'` - 比赛未开始（显示预告面板）
- `'ongoing'` - 比赛进行中（显示实时积分榜，无排名特效）
- `'finished'` - 比赛结束（显示最终积分榜，带排名特效）
